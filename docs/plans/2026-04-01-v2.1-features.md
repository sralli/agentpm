# Agendum v2.1 Implementation Plan

## Task 1: Self-Documenting Tool Descriptions

Rewrite all 11 tool @mcp.tool() description strings to include when to call, what it returns, and what to call next. Each description should teach the workflow for harnesses that have no other guidance (Cursor, Windsurf, Cline).

**Files:** src/agendum/tools.py
**Acceptance Criteria:**
- All 11 tools have descriptions with "When:", "Returns:", "Next:" sections
- Descriptions are plain text (no markdown-only formatting)
- Existing tests pass unchanged

## Task 2: Workflow Hints in Tool Output

Every tool response ends with a `> Next:` hint line guiding the agent to the logical next step. pm_init hints pm_project, pm_project hints pm_ingest or pm_add, pm_ingest hints pm_next, pm_next hints pm_done, pm_done hints pm_next or pm_status.

**Files:** src/agendum/tools.py
**Depends:** Task 1
**Acceptance Criteria:**
- All 11 tools append a Next: hint to their return string
- Hints are contextual (pm_done with unblocked items mentions them)
- Existing tests updated to account for new output suffix

## Task 3: Zero-Config First Run

Auto-initialize .agendum/ directory on first tool call if it doesn't exist. pm_init becomes optional. Default board name derived from git repo name or current directory name.

**Files:** src/agendum/server.py, src/agendum/store/project_store.py, src/agendum/config.py
**Acceptance Criteria:**
- Calling pm_project("create", "foo", "bar") without prior pm_init works
- Default board name matches git remote name or dirname
- pm_init still works for explicit naming
- New test: test_zero_config_first_run

## Task 4: MCP Instructions Field

Add instructions string to FastMCP server configuration with workflow overview. Ignored by harnesses that don't support it.

**Files:** src/agendum/server.py
**Acceptance Criteria:**
- FastMCP server has instructions field with 6-step workflow
- Server starts without errors
- Instructions field is a plain string under 500 chars

## Task 5: Project-Scoped Learnings

Add project parameter to pm_learn (optional). Store project learnings in projects/<project>/learnings/. Add ProjectLearningsSource to enrichment pipeline. Add learnings parameter to pm_done for auto-creating project-scoped learnings.

**Files:** src/agendum/tools.py, src/agendum/store/learnings_store.py, src/agendum/enrichment/sources.py, src/agendum/enrichment/pipeline.py
**Acceptance Criteria:**
- pm_learn with project param stores in projects/<project>/learnings/
- pm_learn without project param stores in global learnings/ (unchanged)
- pm_done with learnings param creates project-scoped learnings
- pm_next enrichment includes ProjectLearningsSource
- New tests for project-scoped learnings CRUD and enrichment

## Task 6: Complexity Signals in Work Packages

Add complexity field to WorkPackage (trivial/small/medium/large). Compute from key_files count, acceptance_criteria count, and dependency count. Include in pm_next output header.

**Files:** src/agendum/models.py, src/agendum/tools.py
**Acceptance Criteria:**
- WorkPackage has complexity and estimated_scope fields
- Complexity computed automatically in pm_next
- pm_next output includes "Complexity: medium (4 files, 3 criteria)"
- New test: test_complexity_signals

## Task 7: Adaptive Enrichment Budget

Budget scales with complexity: trivial 4000, small 6000, medium 8000, large 10000. Budget dict passed to enricher.enrich().

**Files:** src/agendum/enrichment/pipeline.py, src/agendum/tools.py
**Depends:** Task 6
**Acceptance Criteria:**
- enrich() accepts budget dict parameter with per-field limits
- Trivial tasks get less context, large tasks get more
- Default budget (no param) matches current behavior
- Existing enrichment tests updated

## Task 8: Verification Gate in pm_done

Add verified (bool) and verification_notes (str) parameters to pm_done. Unverified completions noted in progress. pm_next hints to verify before calling pm_done.

**Files:** src/agendum/tools.py, src/agendum/models.py
**Acceptance Criteria:**
- pm_done(verified=True) records verification in progress
- pm_done(verified=False) adds "Unverified completion" note
- BoardItem has verified field persisted in frontmatter
- New test: test_verification_gate

## Task 9: Auto-Extract from Git in pm_done

When auto_extract=True and manual decisions/patterns are empty, read git diff stat and last commit message. Store as structured completion record in progress entry.

**Files:** src/agendum/tools.py, src/agendum/env_context.py
**Acceptance Criteria:**
- get_git_diff_stat() and get_last_commit_message() helpers in env_context.py
- pm_done with auto_extract=True captures git state
- Manual decisions/patterns still override auto-extraction
- Graceful fallback when not in a git repo
- New test: test_auto_extract_git

## Task 10: Harness Detection and Multi-Harness Config

Read clientInfo.name from MCP initialize if FastMCP exposes it. Add copy-paste config snippets to README for Claude Code, Cursor, Windsurf, VS Code, Cline, Roo Code.

**Files:** src/agendum/server.py, README.md
**Acceptance Criteria:**
- Server captures clientInfo if available (non-breaking if not)
- README has config snippets for 6+ harnesses
- Each snippet is copy-paste ready with uvx agendum command
